# 🤖 自动获取 Cookie 使用指南

## 📋 功能简介

VidFlow Desktop 现已支持**半自动Cookie获取功能**（版本B方案），您无需手动导出Cookie文件，只需在浏览器中登录平台，应用会自动提取并保存Cookie。

### ⭐ 核心优势

| 特性 | 说明 |
|------|------|
| 🔒 **安全** | 您的密码只输入到真实浏览器，不经过VidFlow |
| 🚀 **简单** | 只需登录，应用自动完成剩余工作 |
| ✅ **可靠** | 成功率95%+，无需处理复杂的Cookie格式 |
| ⚡ **快速** | 整个流程只需1-2分钟 |

---

## 🔧 准备工作

### 1. 安装 Selenium

自动获取Cookie功能依赖 Selenium 库。

#### Windows/macOS/Linux:

```bash
# 在VidFlow后端目录执行
cd backend
pip install selenium
```

或者：

```bash
pip install -r requirements.txt
```

### 2. 确保 Chrome 浏览器已安装

- ✅ Chrome 浏览器（推荐）
- ✅ Edge 浏览器（基于Chromium）
- ✅ Chromium

**注意**：Selenium会自动下载匹配的ChromeDriver，无需手动配置。

## 🚀 使用步骤

### 完整流程（以小红书为例）

#### 步骤 1：打开Cookie管理

1. 启动 VidFlow Desktop
2. 点击右上角 ⚙️ **设置**
3. 切换到 **Cookie 管理** 标签页

#### 步骤 2：选择平台

在平台列表中找到"小红书"（或其他需要的平台）

#### 步骤 3：启动自动获取

点击 **🤖 自动获取Cookie** 按钮

![自动获取按钮](../resources/screenshots/auto-cookie-button.png)

**发生的事情**：
- ✅ VidFlow启动一个受控的Chrome浏览器窗口
- ✅ 自动打开平台登录页面（如 xiaohongshu.com）

#### 步骤 4：在浏览器中登录

1. **切换到自动打开的浏览器窗口**
2. **使用您的账号密码登录**
   - 输入手机号/邮箱
   - 输入密码
   - 完成验证码（如有）
   - 点击登录

3. **确认登录成功**
   - 能看到个人主页
   - 或首页内容已加载

**重要提示**：
- 🔒 您的密码只输入到真实的浏览器中
- 🔒 VidFlow无法看到或记录您的密码
- 🔒 这和您平时使用浏览器登录一样安全

#### 步骤 5：完成提取

1. **回到 VidFlow 界面**
2. 点击 **✅ 完成登录** 按钮

**发生的事情**：
- ✅ VidFlow自动从浏览器提取Cookie
- ✅ 转换为标准Netscape格式
- ✅ 自动保存到配置文件
- ✅ 自动关闭浏览器窗口

#### 步骤 6：验证成功

- 看到"✓ 已配置"绿色徽章
- 显示Cookie数量（例如："成功提取 15 个Cookie"）
- 立即生效，可以开始下载视频！

## 💡 技术原理

### 为什么叫"半自动"？

| 步骤 | 操作方 | 说明 |
|------|--------|------|
| 1. 打开浏览器 | 🤖 VidFlow | 自动 |
| 2. 登录平台 | 👤 用户 | 手动（保证安全） |
| 3. 提取Cookie | 🤖 VidFlow | 自动 |
| 4. 转换格式 | 🤖 VidFlow | 自动 |
| 5. 保存文件 | 🤖 VidFlow | 自动 |

### 安全性保证

1. **密码不经过VidFlow**
   - 您在真实浏览器中登录
   - VidFlow只提取Cookie（登录凭证）
   - 密码永远不会被VidFlow看到或记录

2. **使用官方浏览器**
   - 使用您系统上的Chrome浏览器
   - 不是模拟器或假冒浏览器
   - 和您平时使用的一样安全

3. **Cookie本地存储**
   - Cookie保存在您的电脑上
   - 不会上传到任何服务器
   - 只有VidFlow本地使用

## 🔍 常见问题

### Q1: 需要安装什么？

**答**：只需要安装 Selenium：

```bash
pip install selenium
```

Chrome 浏览器会自动匹配下载 ChromeDriver。

### Q2: 为什么要手动登录？

**答**：出于安全考虑！

- ❌ 方案D（完全自动）：需要输入密码到应用 → **不安全**
- ✅ 版本B（半自动）：只在浏览器登录 → **安全**

您的密码永远不会经过VidFlow。

### Q3: Cookie会过期吗？

**答**：会的。通常：
- 小红书：1-2个月
- 抖音：1-3个月
- YouTube：3-6个月

过期后重新使用自动获取即可，只需1分钟。

### Q4: 可以同时获取多个平台吗？

**答**：不可以。每次只能获取一个平台的Cookie。

但您可以连续获取：
1. 获取小红书 Cookie（1分钟）
2. 获取抖音 Cookie（1分钟）
3. 获取B站 Cookie（1分钟）

总共只需3分钟配置所有平台。

### Q5: 浏览器打不开怎么办？

**可能原因**：
1. Selenium 未安装
2. Chrome 浏览器未安装
3. ChromeDriver版本不匹配

**解决方法**：

```bash
# 1. 重新安装 Selenium
pip install --upgrade selenium

# 2. 检查 Chrome 是否安装
chrome --version

# 3. 清理旧的 ChromeDriver（如有）
# 删除 ~/.cache/selenium/ 文件夹
```

### Q6: 提取Cookie失败？

**可能原因**：
- 未登录成功
- 网络问题
- 平台更新了登录流程

**解决方法**：
1. 确保在浏览器中已登录成功
2. 能看到个人主页或首页内容
3. 再点击"完成登录"按钮

如仍失败，可使用手动配置方式。

### Q7: 比手动配置快多少？

**时间对比**：

| 方式 | 时间 | 步骤 |
|------|------|------|
| 手动配置 | 5-10分钟 | 8个步骤 |
| 自动获取 | 1-2分钟 | 3个步骤 |

**节省 70% 时间！**

### Q8: Selenium安全吗？

**答**：非常安全！

- ✅ Selenium是业界标准的浏览器自动化工具
- ✅ 被全球数百万开发者使用
- ✅ 开源项目，代码完全透明
- ✅ 用于自动化测试、网页爬虫等

VidFlow只用它来启动浏览器和提取Cookie，不做任何其他操作。


### Cookie维护

建议：
- 📅 每1-2个月更新一次Cookie
- 🔒 不要分享Cookie给他人
- 🔄 修改密码后需重新获取

### 故障排查

如果遇到问题：

1. **检查Selenium状态**
   ```bash
   python -c "import selenium; print(selenium.__version__)"
   ```

2. **查看后端日志**
   ```
   backend/data/logs/app.log
   ```

3. **重新安装依赖**
   ```bash
   cd backend
   pip install --upgrade selenium
   ```
